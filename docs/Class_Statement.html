Class Statement

Declares the name of a class and defines its implementation.

Syntax

[Abstract] Class name [ [ typeList ] ] [ Is superclass ] [ Does traitList ]
    [ Where genericConstraints ]
    [ statements ]
End Class

Parts

Abstract
Optional. Specifies that this is an abstract class, and is ordinarily used as the parent of a concrete class.
name
Required. An identifier that names the class.
typeList
Optional. One or more identifiers separated by commas and surrounded by square brackets.
superclass
Optional. The name of another class from which this class inherits methods and properties.
traitList
Optional list of traits, each separated by a comma. For details, see Does Clause (Traits).
genericConstraints
Optional. One or more constraints on a generic parameter separated by commas. See Where Clause (Generics) for details.
statements
Optional. The statements that define the members of the class.
End Class
Terminates the Class block.

Remarks

A Class statement defines a data type known as a class. Classes are fundamental building blocks in object-oriented programming (OOP). For more information, see Objects and Classes in ViviFire.

You can place a Class only within certain contexts. These contexts include user and library modules, other classes, Object declarations, and (maybe [TBD]) traits. You cannot place classes within procedures. For more information, see Declaration Contexts and Default Access Levels

Examples

Listing 1
Require IO #out

Class Person
    Dim myName$, myAge
    Constructor name$, age
        myName$ = name$
        myAge = age
    End Constructor
    Method GetName$
        Return myName$
    End Method
    Method GetAge
        Return myAge
    End Method
End Class

Person #a, "Alice", 24
Person #b, "Bob", 42
#out? #a.GetName$? & " is " & #a.GetAge? & $NL
#out? #b.GetName$? & " is " & #b.GetAge? & $NL
End
' Output:
' Alice is 24
' Bob is 42
