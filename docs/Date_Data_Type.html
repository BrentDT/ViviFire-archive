Date Data Type

Holds 64-bit (8-byte) date and time values in the range 1 January 0001 00:00:00–31 December 9999 23:59:59.

Remarks

Default value
When you declare a variable of type Date and leave it uninitialized,, its value defaults to 0000-00-00 00:00:00, which ViviFire considers an invalid date.
Type characters
Date has no associated type characters.

Date literal

Date form
@year/month/day
Time form
@hours:minutes:seconds
Combined form
@year/month/day@hours:minutes:seconds
/
Separates the parts of a date. If you prefer, you can also use dashes (-) or dots (.) instead.
seconds
You can also specify a fraction of a second by placing a decimal point (.) between two sets of digits. The fractional part is limited to seven digits. Any additional digits are ignored.

Type conversions

If you convert a Date value to the String type, ViviFire renders the date as the short date format specified by the user's locale and the time either as a 12- or 24-hour format likewise.

If you convert to a floating-point type (Single, Double, XFP, or Quad), ViviFire renders the value as the number of days since 30 December 1899 with the time represented as the fraction of the day. Note that this is likely to be a lossy conversion. Converting from a floating-point type to the Date type is also possible.

Examples

Dim foo As Date
foo = @2013/12/11@10:09:08
