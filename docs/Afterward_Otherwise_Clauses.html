Afterward...Otherwise Clauses

Executes a particular block of code depending on how a loop exits.

Syntax

loopHead
    [loopBody]
[ Afterward
    [afterStatements] ]
[ Otherwise
    [otherStatements] ]
loopTail

Parts

loopHead
Required head clause for any of the loop constructs: Do, For, For Each, or While.
loopBody
Optional statements that are repeated by the loop.
afterStatements
Optional statements that are executed just once following the last iteration of loopBody, as determined by loopHead.
otherStatements
Optional statements that are executed just once only if loopBody was not executed due to the evaluation of loopHead.
Not permitted in the post-conditional form of Do...Loop.
loopTail
Required tail clause for its respective loopHead.

Remarks

If both Afterward and Otherwise are used in the same loop, they must appear in that order.

Afterward clause

If you provide an Afterward clause inside a loop construct, the statements in this block of code are executed only when the loop terminates normally. The statements are executed at most once per loop, almost like any code outside of a loop.

If a loop is terminated prematurely using either the Exit or Return statements, the Afterward clause is skipped entirely, and execution continues where appropriate for the given statement.

Otherwise clause

If you provide an Otherwise clause, the statements in this block of code are executed only when the body of the loop was not executed at all. The following table summarizes the conditions under which this can occur.

Loop constructLoop body not executed
 
While condition
When condition is initially false
Do While condition
When condition is initially false
Do Until condition
When condition is initially true
For c = start To end [ Step step ]
When start > end for a positive step, or start < end for a negative step
For Each element In group
When group = #Null or is empty

The Otherwise clause cannot appear in the post-conditional form of Do...Loop because this form always executes the body of the loop at least once.

Scope

The scope of identifiers inside these clauses is the same as the loop in which they appear. This means that any variable declared inside them does not exist outside of that loop.
