Metacharacters and Metastrings

Metacharacters

MetacharacterMeaningCode point
`CRCarriage return13
`ESCEscape27
`FFForm feed12
`LFLine feed10
`SPCSpace32
`TABHorizontal tab9

A non-printable character can be represented in a string literal using a metacharacter, a sequence of characters effectively between two string literals. Because the double quotation mark indicates both the start and end of a string literal, it is not possible to include any in a string literal without an escape mechanism. You can include a quotation mark by typing two quotation marks in a row, which could be seen as two string literals shoved together. Other metacharacters are available that utilize this view. The following table summarizes the available metacharacters.

MetacharacterMeaningExample
""Double quotation mark (", ASCII 34, U+0022)
"I said, ""ViviFire rocks!"""
"/" or "\"The OS-dependent path separator (e.g \ on Windows)
"data"/"datafile.dat"

Metastrings

Metastrings permit the use of nonprintable and/or platform-dependent character sequences directly in your code. They can take different forms, as shown below. Each begins with a $ sign.

$​decimal
$B​binary
$H​hexadecimal
$O​octal
$​mnemonic
decimal
A decimal (base-10) integer in the range 0–65535.
binary
A binary (base-2) integer in the range 0–1111111111111111.
hexadecimal
A hexadecimal (base-16) integer in the range 0–FFFF.
octal
An octal (base-8) integer in the range 0–177777.
mnemonic
An abbreviation for a commonly used string.

Magic metastrings

Certain metastrings have values that change depending on factors such as where you place them. The following table summarizes these "magic" metastrings.

MetastringMeaning
$DATEThe date during compilation in UTC
$FILEThe name of the source file
$LINEThe line number within the source file
$OBJThe name of the class, object, trait, or module
$PROCThe name of the procedure
$TIMEThe time during compilation in UTC
