Method Statement

Defines a method procedure for a class or library.

Syntax

Standard form
[ modifier ] Method name [ parameterList ]
    [ optionalParams ]
    [ statements ]
End Method
Anonymous form
[ modifier ] Method parameterList
    [ statements ]
End Method
Abstract form
Abstract Method signature

Parts

modifier
Optional
  • Abstract Specifies that the method must be overridden by a concrete subclass. Only valid inside an Abstract Class.
  • Override Specifies that the method overrides one inherited from a parent class.
  • Shared Specifies that the method is associated with its surrounding object and not an instance of that object.
name
Required in the standard form.
Not allowed in the anonymous form.
parameterList
Optional in the standard form.
Required in the anonymous form.
optionalParams
Optional in the standard form
Not allowed in the anonymous form
statements
Optional in the standard and anonymous forms.
Not allowed in the abstract form.
End Method
Terminates the Method block.
Not allowed in the abstract form.
signature
The combination of name and parameterList for either a standard or anonymous method.

Remarks

Do not confuse the End Method clause with the Exit Method statement.

A Method is the only kind of procedure directly callable from a Class, Object, or Library. A Function or Sub procedure can only be called by other procedures.
