Property Statement

Declares the names of properties and the procedures called when storing and retrieving values.

Syntax

[modifier] Property [typeList] [!] declarator [ , declarator ]...
  [ Optional ( parameterList ) ]
  [ On getter
    [statements] ]
  [ On setter parameter
    [statements] ]
End Property
or
[modifier] Property [typeList] declarator [ , declarator ]...

Parts

modifier
Optional. One of the following:
  • Abstract specifies that the property contains no implementation and must be overridden in a concrete subclass.
  • Override specifies that the property is a re-implementation of a property inherited from a base class.
  • Shared specifies that the property is owned by the enclosing class and not an instance of that class.
typeList
Optional
!
Optional
declarator
Required. One or more declarative clauses that have the same syntax as the Dim statement, separated by commas.
parameterList
Optional. Allowed only if the property is an array. A list of integer parameters, separated by commas, which hold the array indices supplied by the caller.
getter
A name for the procedure that is called when retrieving the current value of the property. It is effectively a label and has no meaning in the code. "Get" is the standard name for this.
setter
A name for the procedure that is called when storing a new value in the property. It is effectively a label and has no meaning in the code. "Set" is the standard name for this.
parameter
Required after a setter. It must be the same data type as the property. If you do not specify a data type, the type of the property is assumed.
statements
Optional
End Property
Required if you specify a getter or setter; otherwise it is optional.
